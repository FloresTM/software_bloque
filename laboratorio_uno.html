¿Cuál es la diferencia entre Internet y la World Wide Web?: El Internet es la red de dispositivos conectadas entre si que permite el intercambio de información y la comunicación entre los dispositivos. La World Wide Web es un "servicio" que funciona gracias al Internet y permite acceder y compartir información mediante páginas web, imágenes, entre otros.
¿Cuáles son las partes de un URL?: La URL cuenta con varias partes, que se pueden describir de la siguiente manera:
  1. Protocolo: es la forma en que se accede al recurso, como "http" o "https".
  2. Nombre de dominio: es el nombre del sitio web, como "google.com" o "facebook.com".
  3. Ruta: es la ubicación específica del recurso dentro del sitio web, como "/search" o "/profile".
  4. Parámetros: son valores opcionales que se utilizan para personalizar la solicitud, como "?q=term" para una búsqueda.
  5. Anclaje: es una ubicación específica dentro de una página web, identificada por "#anchor".
¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?: Los métodos HTTP se utilizan para indicar la acción que se quiere realizar en el servidor web. Algunos metodos son:
  1. GET: solicita un recurso específico del servidor. La respuesta incluye el recurso solicitado y la información de estado de la respuesta.
  2. HEAD: solicita la misma información que GET, pero sin incluir el cuerpo del mensaje. Se utiliza para obtener información sobre un recurso, como su tamaño o fecha de última modificación, sin descargar todo el contenido.
  3. POST: envía datos al servidor para crear o actualizar un recurso. La respuesta incluye la información de estado y, opcionalmente, los datos creados o actualizados.
  4. PUT: actualiza un recurso existente en el servidor con los datos proporcionados en la solicitud. La respuesta incluye la información de estado y, opcionalmente, los datos actualizados.
  5. PATCH: actualiza parcialmente un recurso existente en el servidor con los datos proporcionados en la solicitud. La respuesta incluye la información de estado y, opcionalmente, los datos actualizados.
  6. DELETE: elimina un recurso existente en el servidor. La respuesta incluye la información de estado y, opcionalmente, los datos eliminados.
¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?: Para enviar un formulario HTML, generalmente se utiliza el método HTTP POST. Esto se debe a que el método POST se utiliza para enviar datos al servidor para crear o actualizar un recurso, como en este caso, que se están enviando datos de un usuario y contraseña para autenticación. Además, el método POST envía los datos en el cuerpo de la solicitud, lo que permite enviar más información que con el método GET. El método GET también se puede utilizar para enviar datos de formulario, pero no se recomienda tanto, el método HTTP POST es el más adecuado para enviar formularios HTML que contienen datos sensibles, ya que permite enviar más información que con GET y no expone los datos en la URL.
¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?: HCuando se accede a una página a través de un URL en un navegador web, generalmente se utiliza el método HTTP GET. El método GET se utiliza para solicitar un recurso específico del servidor, como una página web, y la respuesta incluye el recurso solicitado y la información de estado de la respuesta. Al ingresar un URL en un navegador, se envía una solicitud GET al servidor web que aloja la página correspondiente. El servidor procesa la solicitud, busca la página correspondiente y la envía de vuelta al navegador como respuesta.
Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?: EL codigo 200 no significa un error, sino todo lo contrario. Cuando un servidor web devuelve una respuesta HTTP con el código 200, indica que la solicitud del cliente fue procesada correctamente y se devolvió una respuesta exitosa. El código 200 es el código de estado que indica que la solicitud fue exitosa y que se envió la respuesta solicitada.
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?: Como desarrollador web, es importante garantizar que las páginas web estén disponibles y sean accesibles para los usuarios. Si un usuario encuentra un error 404 en una página, es una indicación de que algo ha salido mal y que la página no está disponible. Como resultado, es responsabilidad del desarrollador investigar el problema y corregirlo para que la página web sea accesible nuevamente.
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?: Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500. El código de estado HTTP 500 indica un error interno del servidor, lo que significa que algo ha salido mal en el servidor y no se puede procesar la solicitud. También, estos errores pueden indicar un problema de seguridad potencial, como una vulnerabilidad en el servidor o en la aplicación web.
¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados: Que un atributo HTML5 está o desaprobado significa que se ha desaconsejado su uso y se recomienda utilizar una alternativa. Aunque estos atributos aún funcionan en los navegadores, pueden dejar de funcionar en versiones futuras o ya no ser compatibles con las últimas especificaciones. Algunos ejemplos son:
  1. El atributo "align" en la mayoría de los elementos, como "img", "table", "div", entre otros. En su lugar, se recomienda utilizar CSS para definir la alineación.
  2. Los elementos "frame" y "frameset" que se utilizaban para crear diseños de sitios web con marcos. En su lugar, se recomienda utilizar elementos HTML y CSS modernos para lograr el mismo resultado.
  3. El atributo "bgcolor" que se utilizaba para definir el color de fondo de una página o un elemento. En su lugar, se recomienda utilizar CSS para definir los estilos de fondo.
¿Cuáles son las diferencias principales entre HTML 4 y HTML5?: HTML5 es una versión actualizada de HTML 4 con varias diferencias significativas. Algunas de las diferencias son:
  1. Soporte multimedia mejorado: HTML5 proporciona soporte nativo para la reproducción de video y audio, lo que significa que ya no es necesario utilizar complementos de terceros como Flash para reproducir estos tipos de contenido.
  2. Nuevos elementos y atributos: HTML5 introduce varios nuevos elementos y atributos que mejoran la semántica y la accesibilidad de los sitios web. Por ejemplo, los nuevos elementos "header", "footer" y "section" ayudan a definir la estructura de una página de manera más clara.
  3. Soporte de aplicaciones web: HTML5 introduce una serie de tecnologías que permiten a los desarrolladores crear aplicaciones web que se ejecutan directamente en un navegador sin necesidad de complementos. Estas tecnologías incluyen el almacenamiento local, la API de geolocalización y la API de Canvas.
  4. Mayor flexibilidad: HTML5 es más flexible en términos de sintaxis y estructura de documento, lo que significa que es más fácil escribir y leer el código HTML.
¿Qué componentes de estructura y estilo tiene una tabla?: Una tabla en HTML consta de varios componentes de estructura y estilo, incluyendo el elemento "table", los elementos "thead", "tbody" y "tfoot", el elemento "tr", los elementos "th" y "td", y los atributos "colspan" y "rowspan". Los estilos CSS se pueden utilizar para personalizar el aspecto de la tabla, incluyendo el tamaño de fuente, el color de fondo, el espaciado de celda y la alineación de texto. Los componentes de estructura y estilo permiten al desarrollador crear tablas personalizadas y de aspecto profesional para mostrar datos de manera clara y organizada.
¿Cuáles son los principales controles de una forma HTML5?: HTML5 tiene una variedad de controles para que los desarrolladores elijan el que mejor se adapte a sus necesidades. Algunos ejemplos son:
  1. Cajas de texto: Permiten al usuario ingresar texto en un solo campo.
  2. Áreas de texto: Permiten al usuario ingresar texto en un campo más grande que una caja de texto.
  3. Botones de opción: Permiten al usuario seleccionar una opción entre varias.
  4. Botones de radio: Permiten al usuario seleccionar una única opción de un conjunto de opciones.
  5. Entre otros...

¿Cuál es el ciclo de vida de los sistemas de información?: El ciclo de vida de los sistemas de información es un proceso estructurado que se utiliza para desarrollar, implementar y mantener sistemas de información. Las etapas comunes son: planificación, análisis, diseño, desarrollo, pruebas, implementación y mantenimiento. Es un proceso continuo que evoluciona a medida que el sistema requiere actualizaciones y mejoras.
¿Cuál es el ciclo de desarrollo de sistemas de información? El ciclo de desarrollo de sistemas de información es un proceso estructurado que consta de seis etapas: identificación de problemas o necesidades, análisis de requisitos, diseño del sistema, implementación del sistema, pruebas del sistema y mantenimiento y evolución del sistema. Es un proceso iterativo que implica volver a las etapas anteriores en caso de que se encuentren problemas o se necesiten cambios adicionales. El objetivo principal es garantizar que el sistema de información desarrollado cumpla con los requisitos del usuario, sea eficiente y efectivo, y proporcione un valor agregado para la organización.
